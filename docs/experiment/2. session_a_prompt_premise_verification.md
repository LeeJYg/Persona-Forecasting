# 세션 A 프롬프트: 방법론 핵심 전제 3가지 독립 검증

## 배경

우리 방법론이 작동하려면 다음 세 전제가 모두 성립해야 한다:
- 전제 1: 페르소나 특성에 따라 hidden state가 달라진다 (Persona Sensitivity)
- 전제 2: Attention이 아이템-관련 페르소나에 높은 가중치를 부여한다 (Market Modeling)
- 전제 3: 가중 합산된 표현에서 판매량 수치를 예측할 수 있다 (Quantitative Transfer)

이 세 전제를 **독립적으로** 검증하는 분석을 수행하라. 코드는 **읽기 + 분석 스크립트 작성**만 하고, 기존 코드를 수정하지 마라.

## 검증 1: Persona Sensitivity 검증

### 목표
"소득이 높고 FOODS 카테고리에 관심 있는 페르소나"와 "소득이 낮고 관심 없는 페르소나"가 동일 아이템에 대해 다른 hidden state를 생성하는지 확인한다.

### 방법

exp011에서 추출된 hidden states를 사용한다 (새로 추출하지 마라).

(a) **Persona-Item Interaction 분석:**
- 동일 아이템에 대한 50개 페르소나의 hidden states를 가져온다.
- 이 50개 벡터 간의 pairwise cosine similarity 분포를 구한다.
- 이것을 **여러 아이템**(최소 10개, 고판매/저판매 골고루)에 대해 반복한다.
- 아이템별로 페르소나 간 분산(L2 distance std 등)을 계산한다.

(b) **페르소나 속성별 클러스터링:**
- 50개 페르소나의 속성 중 income level을 기준으로 상위 10명 / 하위 10명을 분리한다.
- 이 두 그룹의 hidden states 평균 간 L2 distance를 아이템별로 측정한다.
- 비교 기준: 랜덤하게 10명/10명 나눈 경우의 L2 distance (100회 반복하여 분포 생성)

(c) **핵심 판단 기준:**
- income 기반 분리의 L2 distance가 랜덤 분리보다 **통계적으로 유의하게** 클 경우 → 전제 1 지지
- 유의하지 않으면 → hidden state가 페르소나 속성을 인코딩하지 못함 → 전제 1 실패

### 필요 데이터
- exp011의 per-persona hidden states (mean-pool 전 원본). 경로 확인 필요.
- 50개 페르소나의 속성 JSON (income 등). 경로 확인 필요.

**만약 per-persona hidden states가 저장되어 있지 않고 mean-pooled만 있다면, 그 사실을 보고하라.** 이 경우 전제 1 검증을 위해 소규모 재추출이 필요하다.

## 검증 2: Attention as Market Modeling 검증

### 목표
exp011의 학습된 Attention head가 아이템 특성에 따라 다른 페르소나에 높은 가중치를 부여하는지 확인한다.

### 방법

(a) **Attention weight 추출:**
- 학습된 Attention+Bottleneck 모델에서 100개 cold items에 대한 attention weight (shape: 100 items × 50 personas)를 추출한다.
- 모델 체크포인트 경로를 확인하고, forward pass에서 attention weight를 반환하도록 분석 스크립트를 작성하라.

(b) **아이템-페르소나 정합성 분석:**
- 아이템의 카테고리(FOODS, HOUSEHOLD, HOBBIES)별로 attention weight 평균을 구한다.
- 각 카테고리에서 가장 높은 가중치를 받은 상위 5개 페르소나가 누구인지 확인한다.
- 이 페르소나의 속성(income, 관심 카테고리 등)이 해당 카테고리와 직관적으로 일치하는지 확인한다.

(c) **Attention 엔트로피 분석:**
- 각 아이템에 대한 attention weight의 엔트로피를 계산한다.
  - 엔트로피가 높으면(uniform에 가까우면) → attention이 페르소나를 구분하지 못함
  - 엔트로피가 낮으면(sparse하면) → 특정 페르소나에 집중 → Market Modeling 작동
- 엔트로피의 분포(mean, std, histogram)를 보고하라.
- 비교 기준: uniform distribution의 엔트로피 = log(50) ≈ 3.91

(d) **핵심 판단 기준:**
- 평균 엔트로피가 3.91보다 유의하게 낮고, 카테고리별 상위 페르소나가 도메인 지식과 일치하면 → 전제 2 지지
- 엔트로피가 uniform에 가깝거나, 모든 아이템에서 같은 페르소나가 상위이면 → 전제 2 실패

## 검증 3: Quantitative Transfer 검증 (warm 내부)

### 목표
warm items 내부에서조차 head가 개별 아이템의 판매량 차이를 포착하는지 확인한다. 이것이 실패하면 cold 전이 이전에 방법론 자체가 작동하지 않는 것이다.

### 방법

(a) **warm validation 예측 분석:**
- exp011의 warm validation 결과(warm_val_MAE)에서 아이템별 예측값과 실제값을 추출한다.
- 산점도: x축 = 실제 주간 판매량, y축 = 예측 주간 판매량 (아이템별 평균 over 17주)
- 완벽하면 y=x 선 위, 평균 수렴이면 수평선

(b) **판매량 분위별 성능:**
- warm 300개를 실제 판매량 기준으로 4분위(Q1~Q4)로 나눈다.
- 각 분위별 MAE를 계산한다.
- 저판매(Q1) 아이템의 MAE가 고판매(Q4)보다 현저히 높으면 → head가 평균에 수렴하고 있다는 증거

(c) **상관관계:**
- 아이템별 평균 실제 판매량과 아이템별 평균 예측 판매량 간의 Pearson/Spearman 상관계수를 계산하라.
- 상관이 높으면(>0.7) → 상대적 순위는 포착 → embedding에 아이템 정보 있음
- 상관이 낮으면(<0.3) → embedding이 아이템별 차이를 인코딩하지 못함

(d) **핵심 판단 기준:**
- Spearman > 0.5이고 산점도에서 양의 기울기가 보이면 → 전제 3 부분적 지지 (방향은 맞지만 스케일이 문제)
- Spearman < 0.3이면 → 전제 3 실패 (embedding이 아이템 정보를 담지 못함)

## 출력 요구사항

1. 각 검증의 결과를 수치와 함께 보고하라.
2. 각 전제에 대해 **지지 / 부분적 지지 / 실패** 중 하나로 판정하라.
3. 검증 1의 per-persona hidden states 존재 여부를 반드시 확인하고 보고하라.
4. 모든 분석 스크립트를 `scripts/exp012_premise_verification.py`로 저장하라.
5. 결과를 `docs/diagnosis/premise_verification_report.md`에 저장하라.
6. 시각화(산점도, 히스토그램 등)는 `experiments/exp012_premise_verification/figures/`에 저장하라.

## 주의사항

- 기존 코드(exp011 등)를 수정하지 마라.
- 새 분석 스크립트만 작성하라.
- 데이터가 없어서 특정 검증이 불가능한 경우, 무엇이 없는지 명시하고 대안을 제시하라. 임의로 생략하지 마라.

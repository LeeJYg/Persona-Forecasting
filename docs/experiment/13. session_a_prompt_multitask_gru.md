# 세션 A 프롬프트: Multi-Task GRU 구현 및 실험 (exp023)

## 목적

현재 AttnBottleneck head를 **Multi-Task GRU**로 교체하여 스케일 붕괴와 시간 flat 문제를 동시에 해결한다.

**핵심 변경:**
- 단일 regression head → GRU + Direction Head(3-class) + Magnitude Head(regression)
- 시간 무관 예측 → week embedding으로 시간 정보 주입
- raw 타겟 → log(1+sales) 타겟
- 앵커 없음 → price + category_avg를 스케일 앵커로 주입

**기존 데이터 재사용:** warm_raw.pt(300×50×5120), cold_raw.pt(100×50×5120) 그대로 사용. Embedding 재추출 없음.

## 1. 모델 아키텍처 (아래 PyTorch 코드를 그대로 구현하라)

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class ColdStartForecaster(nn.Module):
    def __init__(self, persona_dim=5120, gru_hidden_dim=64, week_emb_dim=16, dropout=0.3):
        super().__init__()
        self.gru_hidden_dim = gru_hidden_dim
        
        # 1. Persona Attention
        self.attn_weights = nn.Linear(persona_dim, 1)
        self.persona_projector = nn.Linear(persona_dim, gru_hidden_dim)
        
        # 2. Temporal Components
        self.week_embeds = nn.Embedding(16, week_emb_dim)
        self.gru = nn.GRU(week_emb_dim, gru_hidden_dim, batch_first=True)
        
        # 3. Direction Head (3-class: UP, DOWN, FLAT)
        self.direction_head = nn.Sequential(
            nn.Linear(gru_hidden_dim, 32),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(32, 3)
        )
        
        # 4. Magnitude Head (Regression)
        self.magnitude_head = nn.Sequential(
            nn.Linear(gru_hidden_dim + 3 + 2, 32),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(32, 1)
        )

    def forward(self, persona_embeddings, scale_anchors, noise_scale=0.0):
        batch_size = persona_embeddings.size(0)
        
        if self.training and noise_scale > 0:
            noise = torch.randn_like(persona_embeddings) * noise_scale
            persona_embeddings = persona_embeddings + noise
        
        attn_scores = F.softmax(self.attn_weights(persona_embeddings), dim=1)
        context = torch.sum(attn_scores * persona_embeddings, dim=1)
        
        h0 = self.persona_projector(context).unsqueeze(0)
        
        week_idx = torch.arange(16, device=persona_embeddings.device).repeat(batch_size, 1)
        week_inputs = self.week_embeds(week_idx)
        
        gru_out, _ = self.gru(week_inputs, h0)
        
        dir_logits = self.direction_head(gru_out)
        
        anchors_expanded = scale_anchors.unsqueeze(1).repeat(1, 16, 1)
        mag_input = torch.cat([gru_out, dir_logits, anchors_expanded], dim=-1)
        
        mag_preds = self.magnitude_head(mag_input).squeeze(-1)
        
        return dir_logits, mag_preds
```

## 2. 데이터 준비

### 2-1: Scale Anchors 계산

각 아이템에 대해 scale_anchors = [price, log(1 + cat_avg)]를 계산하라.

**Warm items:** sell_prices.csv에서 각 warm 아이템의 테스트 기간 평균 가격을 구한다.
**Cold items:** 동일하게 sell_prices.csv에서 구한다 (가격은 cold-start에서도 알 수 있는 정보).
**cat_avg:** 같은 cat_id의 warm 아이템들의 y_warm 주간 평균 판매량의 평균. **cold 아이템의 판매량을 사용하지 마라.**

```python
# cat_avg 계산 예시
for cat in ['FOODS', 'HOBBIES', 'HOUSEHOLD']:
    warm_cat_items = warm_meta[warm_meta.cat_id == cat]
    cat_avg[cat] = y_warm[warm_cat_items.index].mean()
```

### 2-2: Direction Labels 생성

warm 아이템의 y_warm (300, 16)에서 direction labels를 생성한다.

```python
# threshold = 5%: 변화율이 5% 미만이면 FLAT
diff = y_warm[:, 1:] - y_warm[:, :-1]  # (300, 15)
pct_change = diff / (y_warm[:, :-1] + 1e-6)  # 0 방지

direction = torch.zeros_like(diff, dtype=torch.long)  # 0=FLAT
direction[pct_change > 0.05] = 0   # UP
direction[pct_change < -0.05] = 1  # DOWN
direction[torch.abs(pct_change) <= 0.05] = 2  # FLAT
```

**주의:** direction의 shape은 (300, 15)이다 — 16주에서 15개 전환.
따라서 GRU의 direction head 출력도 week 2~16에 대해서만 direction을 예측해야 한다.
또는 week 1의 direction은 "이전 주 대비"가 없으므로 FLAT으로 설정.

**이 부분의 구현 방식을 선택하라:**
- 방법 A: GRU 출력 16주 중 2~16번째(15개)에 대해서만 direction loss 계산
- 방법 B: week 1도 포함하되 FLAT 라벨 부여

방법 A를 추천한다.

### Phase 2: Magnitude Only (direction_head freeze)

```python
for param in model.direction_head.parameters():
    param.requires_grad = False
optimizer = Adam(filter(lambda p: p.requires_grad, model.parameters()), lr=1e-3, weight_decay=1e-4)

for epoch in range(200):
    _, mag_preds = model(X_warm, anchors_warm, noise_scale=0.01)
    loss = F.l1_loss(mag_preds, y_warm_log)
    loss.backward()
    clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
```

### Phase 3: Joint Fine-tuning

```python
for param in model.parameters():
    param.requires_grad = True
optimizer = Adam(model.parameters(), lr=1e-5, weight_decay=1e-4)
lambda_dir, lambda_mag = 0.4, 0.6

for epoch in range(100):
    dir_logits, mag_preds = model(X_warm, anchors_warm, noise_scale=0.01)
    dir_loss = CrossEntropyLoss(dir_logits[:, 1:].reshape(-1, 3), gt_directions.reshape(-1))
    mag_loss = F.l1_loss(mag_preds, y_warm_log)
    loss = lambda_dir * dir_loss + lambda_mag * mag_loss
    loss.backward()
    clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
```

## 5. Ablation 설계

| 변형 | 설명 | 목적 |
|------|------|------|
| **full** | Multi-Task GRU (위 전체) | 최종 모델 |
| **no_direction** | magnitude head만, direction head 제거 | direction의 기여 확인 |
| **no_gru** | GRU 없이, 기존 AttnBottleneck + scale anchor + log target | GRU의 기여 확인 |
| **no_anchor** | Multi-Task GRU에서 scale_anchors 제거 | 앵커의 기여 확인 |
| **anchor_only** | embedding 없이 scale_anchors만으로 GRU 학습 | embedding의 기여 확인 |
| **baseline** | 기존 G1 (AttnBottleneck, residual, raw target) | 현재 최선 |

모든 변형: 3 seeds (42, 123, 777)

### anchor_only 구현:
persona_embeddings를 zeros로 대체 (또는 attention을 bypass하고 h0를 zeros로 설정).
GRU는 week_embedding만으로 동작하고, magnitude head는 scale_anchors에만 의존.

## 주의사항

- cold 평가는 16주, evaluate_weekly 함수 사용.
- scale_anchors의 cat_avg는 warm 300개에서만 계산. cold 정보 사용 금지.
- log target 사용: y_train = log1p(y_warm), 추론 후 expm1로 역변환.
- direction labels에서 y_warm이 0인 주의 pct_change 처리: 분모에 1e-6 추가.
- 3 seeds 평균±std로 보고. 단일 seed 결과로 결론 내지 마라.
- direction head의 3-class accuracy도 warm validation에서 보고하라 (방향 예측 자체의 성능).
- **기존 코드(exp016 등)를 수정하지 마라. 새 스크립트를 작성하라.**
- **anchor_only에서 embedding을 전혀 사용하지 않는다는 것을 반드시 확인하라.**

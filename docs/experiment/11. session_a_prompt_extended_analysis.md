## 파트 4: 복수 Cold Set 실험

### 배경

현재 cold 100개는 seed=42 하나의 층화 샘플링 결과이다.
같은 층화 조건(카테고리 균등, 티어 가중치)으로 seed를 바꿔 여러 cold set을 만들고,
Track B와 LightGBM의 상대 성능이 일관되는지 확인해야 한다.

### 방법

**단, 현재 embedding이 추출된 아이템은 warm 300개 + cold 100개 = 총 400개뿐이다.**
추가 아이템의 embedding 추출 없이 할 수 있는 방법:

### 방법

**현재 embedding이 추출된 400개(warm 300 + cold 100) 내에서 재분할하는 것은 불가능하다.**
warm 300개는 head 학습 시 판매량을 타겟으로 사용했으므로, 이를 cold로 재분류하면 data leakage가 발생한다.

따라서 복수 cold set 실험을 하려면 **새로운 cold 아이템의 embedding을 추출해야 한다:**

**추가 cold 아이템 embedding 추출로 cold set 확장**
- CA_1의 나머지 ~2,649개 아이템 중에서 cold를 추가로 뽑으려면 **해당 cold 아이템의 embedding만 추출하면 된다.** 
- head는 기존 300 warm으로 학습된 것을 그대로 사용한다. warm을 추가로 뽑을 필요 없다.
- **추가 cold 200개 × 50 페르소나 = 10,000 텍스트의 embedding 추출 소요 시간을 추정하라.**
  (exp011에서 20,000 텍스트에 ~20시간 → 10,000 텍스트 ≈ ~10시간 추정. 실제 확인 필요.)
- LightGBM은 embedding 없이 실행 가능하므로 추가 cold에 대한 비교 예측값은 바로 생성 가능.
- **seed를 바꿔(예: seed 0, 1, 2, 3, 4) 매번 다른 200개를 뽑으면 5개 cold set을 만들 수 있다.**
  단, 5 × 200 = 1,000개를 모두 뽑으면 50,000 텍스트 ≈ ~50시간. 비용 대비 효과를 보고하라.

**이번 프롬프트에서는 embedding 추출을 실행하지 않는다.** 소요 시간 추정과 실현 가능성만 보고하라.

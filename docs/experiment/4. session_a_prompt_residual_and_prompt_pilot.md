# 세션 A 프롬프트: Residual Scaling Law + 프롬프트 파일럿 (exp013 후속)

## 배경

exp013 결과에서 핵심 미해결 질문이 두 가지 남았다:

1. **C-1 residual에서 MAE가 동일한 이유**: 원본과 residual이 같은 MAE(58.7 vs 58.4)를 보인다.
   이것이 "방향 정보가 원래 불필요하다"(해석 1)인지, "데이터 부족으로 새 정보를 활용 못 한다"(해석 2)인지 구분해야 한다.

2. **프롬프트 변형의 효과**: V4-A/V4-B가 원본 수준에서 코사인 유사도를 낮출 수 있는가?

---

## 실험 D: Residual Embedding Scaling Law

### 목적
실험 A와 동일한 scaling law를 **residual embedding**으로 반복한다.
원본에서는 100개 이후 정체했는데, residual에서도 같은 패턴이면 해석 1(방향 정보 불필요)이 지지된다.
residual에서 정체 없이 계속 개선되면 해석 2(데이터 부족으로 새 정보 미활용)가 지지된다.

### 설계

exp013 실험 A와 **완전히 동일한 설계**(동일 train/test split, 동일 seed 3개)를 사용하되,
입력 embedding만 residual(item-mean subtracted)로 교체한다.

```python
item_mean = warm_raw.mean(dim=1, keepdim=True)
warm_residual = warm_raw - item_mean
```

측정 지표도 동일: test MAE, entropy, P042 weight, 카테고리 분화율.

### 핵심 판단 기준

- residual에서 MAE가 N=250까지 **계속 감소**하면 → 해석 2 지지 → warm 확장 + residual 조합이 유망
- residual에서도 100 이후 정체하면 → 해석 1 지지 → 방향 정보 추가가 무의미 → 근본적으로 embedding 자체의 정보량이 부족

원본 실험 A 결과와 **나란히 비교하는 테이블**을 작성하라:

| Train N | 원본 MAE | Residual MAE | 원본 Entropy | Residual Entropy |
|---------|---------|-------------|-------------|-----------------|
| 30 | 71.9 | ? | 0.116 | ? |
| ... | ... | ... | ... | ... |

## 실험 E: C-2 프롬프트 파일럿 실행

### 목적
V4-A(Persona-Heavy)와 V4-B(Contrastive)가 embedding의 코사인 유사도를 낮추는지 확인한다.

### 설계

exp013 보고서에 이미 V4-A, V4-B 프롬프트가 설계되어 있다. 이를 사용한다.
**스크립트 `scripts/exp013_scaling_and_direction.py`에 `--run-c2` 플래그가 이미 구현되어 있으므로, 이를 실행하라.**

```bash
CUDA_VISIBLE_DEVICES=0,1,2,3 \
HF_HOME=/mnt/sdd1/jylee/huggingface_cache \
conda run --no-capture-output -n persona-forecasting \
  python scripts/exp013_scaling_and_direction.py --run-c2
```

만약 `--run-c2` 구현이 불완전하거나 오류가 발생하면, 오류 내용을 보고하고 수정하라.

### 측정 (5개 아이템 × 50 페르소나)

| 지표 | V3 (기존) | V4-A | V4-B |
|------|----------|------|------|
| 페르소나 pairwise cosine sim (평균) | 0.978 | ? | ? |
| income L2 distance | 164.7 | ? | ? |
| PCA top-1 PC 설명 비율 | 0.734 | ? | ? |
| PCA top-10 PC 설명 비율 | 0.978 | ? | ? |

### 핵심 판단 기준

- V4-A 또는 V4-B에서 코사인 유사도가 **0.95 이하**로 떨어지면 → 전체 재추출의 가치 있음
- 0.97 이상으로 유지되면 → 프롬프트 수준의 변경으로는 한계 → 모델/레이어/추출 방식 변경 필요
- V4-A와 V4-B 중 더 효과적인 쪽을 식별하라

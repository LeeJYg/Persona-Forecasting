## 파트 A: Residual vs 원본 MAE 차이 원인 규명

### 배경
- exp011 (원본 embedding, 17주): cold MAE **67.19** (evaluate_weekly 기준) 
- exp016 G1 (residual embedding, 16주): cold MAE **8.77**
- 이 차이의 원인이 밝혀지지 않았다. residual 때문인지, 평가 방식 때문인지, 다른 요인 때문인지 분리해야 한다.

### 작업

**A-1: 동일한 평가 함수(evaluate_weekly)로, 동일한 16주 기준으로, 다음 4가지 변형의 cold 예측값을 생성하고 평가하라.**

| 변형 | Embedding | 학습 코드 | Epoch | 평가 |
|------|-----------|----------|-------|------|
| A-1a | 원본 raw | exp016 AttnBottleneck + train_head | 500 | evaluate_weekly, 16주 |
| A-1b | residual | exp016 AttnBottleneck + train_head | 500 | evaluate_weekly, 16주 |
| A-1c | 원본 raw | exp011의 학습 코드 그대로 | exp011 설정 | evaluate_weekly, 16주 |
| A-1d | residual | exp011의 학습 코드 그대로 | exp011 설정 | evaluate_weekly, 16주 |

A-1c, A-1d를 위해 exp011의 학습 코드(run_exp011.py 또는 해당 스크립트)를 확인하고, 
exp016의 AttnBottleneck과 **어떤 점이 다른지** 코드 레벨에서 비교하라.
차이점을 목록으로 보고하라 (함수명, 파라미터, loss function, optimizer 등).

**A-2: 각 변형의 cold 예측값 분포를 보고하라.**

| 변형 | pred mean | pred std | pred min | pred max |
|------|-----------|---------|---------|---------|

비교 기준:
- y_cold actual: mean=10.26, std=28.63
- y_warm actual: mean=96.23, std=120.35

원본 embedding의 예측이 y_warm 분포에 가깝고 residual이 y_cold에 가까우면 → 스케일 차이가 원인.
둘 다 비슷하면 → 다른 요인.

**A-3: 아이템별 scatter plot 데이터를 제공하라.**

| item_id | actual_mean | pred_raw_mean | pred_residual_mean |
|---------|------------|---------------|-------------------|

(100개 전체, CSV로도 저장)

## 파트 B: 아이템별 성적표 + 메타 정보

### B-1: Cold 100개 아이템 성적표

**Track B (exp016 G1 설정: residual, 16주, 500 epoch, seed=42 고정)**로 예측.
**lightgbm_proxy_lags, knn_analog**의 예측값도 필요.

exp006의 competitor 예측값이 저장된 경로를 확인하라. 
없으면 exp006 코드를 재실행하여 예측값을 확보하라.
**예측값의 저장 경로를 보고하라.**

| 필드 | 설명 |

| trackb_DirAcc | Track B 아이템별 DirAcc (flat 포함) |
| trackb_DirAcc_noflat | Track B 아이템별 DirAcc (flat 제외, 보조) |
| lgbm_MAE | lightgbm 아이템별 MAE |

## 파트 C: 맥락 검증 자료

### C-1: 평가 함수 소스 코드

현재 사용 중인 evaluate_weekly 함수의 **전체 소스 코드**를 보고서에 포함하라.
특히:
- MAE 계산 방식 (아이템별 평균 후 전체 평균? 전체 플랫 평균?)
- DirAcc 계산 방식 (flat 처리)
- WRMSSE 계산 방식 (스케일 팩터 어떻게 계산?)

### C-2: 실험 config 요약

이번 실험에서 사용한 모든 설정을 보고서 상단에 명시:
- Track B: embedding 유형, epoch, lr, weight_decay, bottleneck, seed
- 평가: 주 수, flat 처리 방식
- competitor: 어떤 예측값을 사용했는지 (파일 경로)

### C-3: 이전 실험과의 수치 일관성 확인

exp018에서 MAE 8.39로 보고된 것과 exp019에서 MAE 9.67로 보고된 것의 차이 원인을 확인하라.
- 같은 모델/설정인데 다른 수치가 나왔다면 → seed, 평가 기간, 평가 함수 차이
- 다른 모델/설정이라면 → 무엇이 다른지 명시

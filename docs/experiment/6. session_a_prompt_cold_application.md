# 세션 A 프롬프트: Cold Items 적용 + Rescale Ablation (exp016)

## 배경

exp015에서 warm 내부 LOO 결과:
- K-NN rescale (K=50, mean_pooled): MAE 37.90 — global Attn+BN(55.8) 대비 -32%
- 유사 아이템 접근이 유망하다는 강한 신호

그러나 두 가지 핵심 미확인 사항이 있다:
1. Cold items에 적용했을 때도 효과가 있는가?
2. Rescale의 성능이 embedding(우리 방법론)에서 오는가, K-NN 판매량 통계에서 오는가?

---

## 실험 F: Cold Items 적용

### 설계

warm 300개 전체를 사용하여:
1. Global Attn+Bottleneck head를 학습 (residual embedding 사용, exp014와 동일)
2. cold 100개 각각에 대해, warm 300개 중 K-nearest neighbors를 mean_pooled embedding cosine distance로 찾는다
3. rescale 적용: pred_cold_i = global_head(cold_residual_i) × (mean_sales_of_K_neighbors / mean_sales_of_all_warm)
4. K = [5, 10, 20, 50]

### 비교 대상 (exp006에서 가져옴, 모두 weekly MAE, 100 cold items)

| 모델 | MAE | DirAcc |
|------|-----|--------|
| lightgbm_proxy_lags | 8.48 | 0.343 |
| similar_item_avg | 8.64 | 0.232 |
| Track A calibrated | 8.90 | 0.393 |
| knn_analog | 9.57 | 0.412 |

### 측정
- cold MAE, DirAcc (exp006과 동일한 평가 방식 사용)
- **주 수를 16주로 통일** (exp006과 동일하게 불완전한 주 제거)

### 주의사항 — 평가 기준 통일
exp011은 17주(부분 주 포함)였고 exp006은 16주(완전한 주만)였다.
이번 실험에서는 **반드시 exp006과 동일한 16주 기준**으로 평가하라.
cold_test에서 불완전한 주를 제거하는 로직을 exp006(run_competitors.py)의 _to_weekly 함수에서 확인하고 동일하게 적용하라.

## 실험 G: Rescale Ablation — 우리 방법론의 기여 분리

### 목적
rescale의 성능이 (a) global head(embedding+attention)에서 오는지, (b) K-NN 판매량 통계에서 오는지 분리한다.
이것이 분리되지 않으면, "LLM persona embedding이 기여한다"고 주장할 수 없다.

### 설계

**warm LOO (300개)와 cold (100개) 모두에서** 다음 5가지 변형을 테스트:

| 변형 | 예측 방식 | 의미 |
|------|----------|------|
| G-1: Global head only | global_head(embedding_i) | embedding + attention의 순수 기여 |
| G-2: Global head + rescale | G-1 × (K-NN mean sales / global mean sales) | exp015의 방식 |
| G-3: Constant + rescale | **global_mean_sales** × (K-NN mean sales / global_mean_sales) = K-NN mean sales | head 없이 K-NN 판매량 평균만 사용 |
| G-4: Random embedding + rescale | random_head(random_embedding_i) × rescale | embedding이 랜덤이어도 rescale이 작동하는지 |
| G-5: K-NN weighted avg | Σ(w_j × sales_j) / Σ(w_j), w=1/distance | 순수 K-NN 가중 평균 (exp015 검증 2) |

K=50으로 고정 (exp015에서 최적이었으므로).

### G-4 구현 방법
- warm_raw와 동일한 shape의 랜덤 텐서 생성 (torch.randn)
- 이 랜덤 embedding으로 K-NN을 찾고 rescale 적용
- 3회 반복하여 평균

### 핵심 판단 기준

| 시나리오 | 의미 |
|---------|------|
| G-2 << G-3 | head가 rescale 위에 추가 정보를 제공 → 우리 방법론 기여 확인 |
| G-2 ≈ G-3 | head의 기여 없음, K-NN 판매량 평균이 전부 → 우리 방법론 가치 미미 |
| G-2 < G-5 | rescale이 K-NN 가중 평균보다 나음 → head의 방향 정보가 rescale과 시너지 |
| G-4 ≈ G-2 | 랜덤 embedding에서도 rescale이 작동 → K-NN이 embedding 품질과 무관 |
| G-4 >> G-2 | embedding 품질이 K-NN 이웃 선택에 영향 → embedding의 실질 기여 확인 |

**가장 나쁜 시나리오:** G-2 ≈ G-3 ≈ G-4 → rescale이 전부이고 embedding은 무의미
**가장 좋은 시나리오:** G-2 << G-3, G-2 << G-4 → head와 embedding 모두 기여

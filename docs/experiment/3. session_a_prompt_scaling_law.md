# 세션 A 프롬프트: Warm 내부 Scaling Law 및 분포 불일치 검증 (exp013)

## 배경 및 목적

exp012에서 전제 2(Attention Market Modeling)가 부분적으로 실패했다. 구체적으로:
- Attention entropy는 낮지만(0.303), P042가 모든 카테고리에서 지배적
- 카테고리별 페르소나 분화가 없음 → "시장 모델링"이 아닌 "특정 페르소나 편향"

이 실패의 원인이 **학습 데이터 부족(300개)**인지, **embedding 자체의 정보 부족**인지를 구분해야 한다.
또한 warm/cold 분포 불일치(mean 96 vs 10)가 성능에 미치는 영향도 정량화해야 한다.

**이 실험은 기존 embedding(warm_raw.pt, 300×50×5120)만 사용한다. 새로운 embedding 추출 없음.**

---

## 실험 A: 학습 셋 크기별 Attention 분화 (Scaling Law)

### 설계

warm 300개를 고정 테스트 50개 + 학습 풀 250개로 나눈다.
테스트 50개는 판매량 분포가 전체와 유사하도록 stratified sampling한다 (분위별 균등).

학습 셋 크기를 [30, 50, 75, 100, 150, 200, 250]으로 변화시키며, 
각 크기에서 Attention+Bottleneck head를 학습하고 다음을 측정한다:

| 측정 지표 | 의미 |
|----------|------|
| test MAE | 예측 성능 |
| mean attention entropy | attention 집중도 |
| 카테고리별 top-1 페르소나 일치율 | 카테고리 분화 (아래 정의) |
| P042 평균 weight | P042 지배 정도 |

### 카테고리별 top-1 페르소나 일치율 정의

각 테스트 아이템에 대해 attention weight가 가장 높은 페르소나를 식별한다.
같은 카테고리의 아이템들이 **동일한 top-1 페르소나를 공유하는 비율**을 계산한다.
- 비율이 높으면(>0.8) → 카테고리 내에서는 일관되지만 카테고리 간 차이가 있는지 확인
- 모든 카테고리에서 같은 페르소나가 top-1이면 → 카테고리 분화 실패

추가로, 3개 카테고리(FOODS/HOBBIES/HOUSEHOLD)의 top-1 페르소나가 **서로 다른지**를 확인한다.

### 안정성

각 학습 셋 크기에서 랜덤 시드를 3개 사용하여 반복한다.
결과는 mean ± std로 보고한다.

### 핵심 판단 기준

- 학습 셋 크기가 커질수록 **P042 weight가 감소**하고 **카테고리별 top-1이 분화**하면 → 데이터 부족이 원인 → warm 확장(경로 B) 정당화
- 250에서도 P042 지배가 해소되지 않으면 → embedding 자체의 문제 → 프롬프트/레이어 변경 필요
- 학습 셋 크기가 커져도 test MAE가 정체하면 → head 구조 한계 또는 embedding 정보 부족

## 실험 B: 분포 불일치 영향 정량화

### 설계

warm 300개를 판매량 기준으로 정렬하고, 4분위(Q1~Q4)를 만든다.

| 변형 | 학습 셋 | 테스트 셋 | 목적 |
|------|---------|----------|------|
| B-1 (matched) | Q2+Q3+Q4에서 200개 | Q2+Q3+Q4에서 50개 | 분포 일치 시 성능 |
| B-2 (cold-like) | Q2+Q3+Q4에서 200개 | **Q1 전체 (75개)** | cold와 유사한 분포 불일치 |
| B-3 (reverse) | Q1+Q2+Q3에서 200개 | **Q4 전체 (75개)** | 역방향 불일치 |

각 변형에서 Attention+Bottleneck을 학습하고 test MAE, attention entropy, P042 weight를 측정한다.

### 핵심 판단 기준

- B-2 MAE >> B-1 MAE이면 → 분포 불일치가 성능 저하의 주 원인 → warm 확장 시 cold와 유사한 판매량 아이템을 충분히 포함해야 함
- B-2 MAE ≈ B-1 MAE이면 → 분포 불일치는 주 원인이 아님 → 문제는 다른 곳에 있음
- B-2에서 attention이 카테고리별로 분화하는지도 확인 (분포 불일치가 attention 패턴에 미치는 영향)

## 실험 C: Embedding 방향 분리 (방향 2 탐색)

### 배경

exp012에서 코사인 유사도 0.978은 50개 페르소나의 hidden state가 거의 같은 **방향**을 가리키고 있음을 의미한다.
L2 distance에서는 income 기반 차이가 통계적으로 유의했지만, 이는 방향이 아닌 **크기(norm)** 차이일 수 있다.
방향이 같으면 attention이 페르소나를 구분할 정보가 부족하다.

이 실험은 **embedding 재추출 없이** 현재 데이터에서 방향 분리를 만드는 후처리 방법을 테스트한다.

### 실험 C-1: Item-Mean Subtraction (즉시 실험 가능)

원리: 각 아이템에 대한 50개 페르소나 hidden state에서 **그 아이템의 페르소나 평균**을 뺀다.
남는 것은 순수한 페르소나 간 변동 성분이다.

```python
# warm_raw shape: (300, 50, 5120)
item_mean = warm_raw.mean(dim=1, keepdim=True)  # (300, 1, 5120)
warm_residual = warm_raw - item_mean              # (300, 50, 5120)
```

이 residual embedding에 대해 다음을 측정하라:
1. 페르소나 간 pairwise cosine similarity (exp012 (a)와 동일 방식) — 0.978에서 얼마나 낮아지는가?
2. income 기반 분리 vs 랜덤 분리 L2 distance (exp012 (b)와 동일 방식)
3. PCA: top-1, top-10 PC 설명 비율 — 99.59%에서 얼마나 내려가는가?
4. 이 residual embedding으로 Attention+Bottleneck을 학습했을 때 test MAE, entropy, P042 weight

### 실험 C-2: 프롬프트 변형 파일럿 (소규모, GPU 사용)

**이 실험은 GPU를 사용하여 소규모 embedding을 새로 추출한다.**
기존 exp011 프롬프트를 확인하고, 다음 2가지 변형 프롬프트를 설계한다.

현재 프롬프트의 구조를 먼저 확인하라 (파일 경로를 찾아서 내용을 보고하라).
그 위에서 다음 원칙으로 변형한다:

**변형 V4-A (Persona-Heavy):**
- 아이템 정보를 최소화: 카테고리명 + 가격만 남김
- 페르소나 속성을 서술형으로 확장: "당신은 월 소득 $3,200의 30대 여성으로, 두 아이의 어머니이며, 식료품에 주 예산의 40%를 할당합니다. 취미용품에는 거의 지출하지 않습니다..." 식으로 구체적 서사 부여
- 지시: "이 상품에 대한 당신의 구매 의향과 예상 구매 수량을 1주 기준으로 답하시오"

**변형 V4-B (Contrastive Prompt):**
- 아이템 정보는 현재와 동일하게 유지
- 프롬프트 끝에 대조 지시 추가: "당신과 정반대의 경제적 상황에 있는 소비자라면 이 상품을 어떻게 대할지도 함께 고려하여, 당신만의 고유한 판단을 내리시오"
- 목적: LLM이 페르소나 차이를 내부적으로 더 강하게 활성화하도록 유도

**파일럿 규모:** 5개 아이템(FOODS 2, HOBBIES 2, HOUSEHOLD 1) × 50 페르소나 = 250 텍스트
**측정:** 코사인 유사도, L2 distance, PCA — 기존 V3와 비교

### 핵심 판단 기준

- C-1에서 residual 코사인 유사도가 0.978보다 현저히 낮아지고(예: <0.9), attention 성능이 유지 또는 개선되면 → 후처리만으로 방향 분리 가능 → 가장 효율적인 경로
- C-1에서 residual이 노이즈만 남기면(MAE 악화) → 아이템 평균 성분에 유의미한 페르소나 정보가 섞여 있었다는 뜻 → 후처리 부적합
- C-2에서 V4-A 또는 V4-B가 코사인 유사도를 0.95 이하로 낮추면 → 전체 재추출의 가치가 있음
- C-2에서 효과가 미미하면 → 프롬프트 변형으로는 한계 → 모델/레이어 변경 검토

## 주의사항

- 기존 코드를 수정하지 마라.
- 실험 A, B, C-1은 warm_raw.pt (300×50×5120)과 기존 y_warm만 사용하라.
- 실험 C-2만 GPU를 사용한 소규모 embedding 추출이 필요하다. exp011의 추출 코드를 참조하되, 5개 아이템 × 50 페르소나 = 250 텍스트만 추출하라.
- Attention+Bottleneck의 구조는 exp011과 동일하게 유지하라 (동일 hyperparameter).
- 만약 exp011의 Attention+Bottleneck 모델 구조를 코드에서 확인할 수 없다면, 그 사실을 보고하고 확인 가능한 경로를 탐색하라.
- C-2의 프롬프트 변형을 설계할 때, **현재 프롬프트를 먼저 확인하고 보고한 후** 변형을 작성하라. 현재 프롬프트를 모르는 상태에서 변형을 만들지 마라.

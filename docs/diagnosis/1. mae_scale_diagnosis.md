# MAE 스케일 불일치 원인 규명
**작성일:** 2026-03-10
**대상:** exp006 (competitors, weekly MAE 8~9) vs exp011 (Track B V3, MAE 59~73)

---

## 결론 요약

**단위는 동일하다.** MAE 차이의 원인은 unit 불일치가 아니라, **warm/cold 아이템 판매량 분포의 극심한 격차**이다.
alpha 보정 후 exp011 MAE(9.7~11.4)는 exp006 MAE(8~9)와 비교 가능한 범위에 들어온다.

---

## Step 1: 타겟 변수 추적

### exp006 (run_competitors.py)

**y 생성 경로:**
1. `cold_test = pd.read_csv("data/processed/cold_start/cold_test.csv")` — daily 판매량
2. `cold_test_weekly = _to_weekly(cold_test, "sales")` — ISO-week 기준 일별 sales를 주간 합산
3. 불완전한 주 제거 (7일 미만) → **16 complete weeks**
4. `evaluate_weekly(cold_test_weekly, pred, ...)` → `mae(merged["sales"], merged["pred_sales"])` — (item_id, store_id, date=week_start) 기준 merge 후 MAE

**y 단위:** ISO-week 주간 합산 판매량 (단위: 개/주)
**아이템 수:** 100 cold items
**총 샘플 수:** 100 × 16 = 1,600 (item-week rows)

**실제 y 기초 통계 (코드로 재현):**
```
mean: 10.26, std: 29.25, min: 0.0, max: 432.0
```

### exp011 (run_exp011.py, git commit ed74e17)

**y 생성 경로:**
```python
date_start = str(cold_test["date"].min().date())   # "2016-01-01"
```

```python
sub["week"] = pd.to_datetime(sub["date"]).dt.to_period("W")  # Mon-Sun 기준 ISO 주
```

## Step 2: 동일 기준 정렬

### 단위가 동일한가?

**YES.** 양쪽 모두 주간 합산 판매량이며, 동일한 `cold_test.csv`를 소스로 사용.

| 비교 항목 | exp006 | exp011 |
|-----------|--------|--------|
| y_cold 단위 | ISO-week 주간 합산 | pd.Period("W") 주간 합산 |
| y_cold mean | **10.26** | **9.96** |
| y_cold std | 29.25 | 28.63 |
| y_cold max | 432.0 | 432.0 |
| 주 수 (cold) | 16 (완전한 주만) | 17 (부분 주 포함) |

**결론:** 실질적으로 동일한 분포 (mean 차이 0.30, std 차이 0.62).
주 수 차이(16 vs 17)는 MAE에 미미한 영향만 미친다.

---

### MAE 차이의 실제 원인: warm/cold 분포 불일치

**핵심 발견:**

```
y_warm mean = 96.23   (head 학습에 사용된 warm 300개 아이템)
y_cold mean = 9.96    (head 예측 대상 cold 100개 아이템)
warm / cold ratio = 9.66×
```

**설명:**
Ridge 회귀 헤드는 y_warm(mean≈96.2)을 타겟으로 학습된다.
임베딩이 아이템별 고유 정보를 거의 담지 못하므로(PCA: top-1 PC가 57.9% 설명),
헤드는 warm items의 **평균 판매량 패턴**에 수렴한다.
이 헤드를 cold items(mean≈9.96)에 적용하면 ~10배 과대예측이 발생한다.

| | exp006 MAE | exp011 raw MAE | exp011 α-MAE |
|---|---|---|---|
| Ridge (mean) | 8~9 범위 | **73.32** | **11.38** |
| Attn + Linear | 8~9 범위 | **68.14** | **9.66** |
| Attn + Bottleneck | 8~9 범위 | **67.71** | **10.29** |

**alpha 보정 후 MAE(9.7~11.4)는 exp006 MAE(8~9)와 비교 가능한 범위에 들어온다.**

단, alpha = mean(y_cold) / mean(y_raw_pred)로 계산되는 post-hoc scale 보정이므로,
이 보정의 논문 내 정당성을 명시해야 한다.

## Step 3: 통합 비교 테이블 가능 여부

**단위가 동일하므로 테이블 작성 가능하다. 단, 두 가지 조건을 충족해야 한다:**

1. raw MAE와 α-MAE를 구분해서 표시할 것
2. 불완전한 주(1주) 처리 방식 통일 필요 (exp006: 제거, exp011: 포함)

| 모델 | MAE (weekly, 100 cold items) | DirAcc | 비고 |
|------|-------------------------------|--------|------|
| lightgbm_proxy_lags | 8.48 | 0.343 | exp006 (16주) |
| similar_item_avg | 8.64 | 0.232 | exp006 (16주) |
| Track A calibrated | 8.90 | 0.393 | exp006 (16주) |
| knn_analog | 9.57 | 0.412 | exp006 (16주) |
| **Track B V3 Attn+Lin** | **raw 68.14 / α 9.66** | **0.524** | exp011 (17주) |
| **Track B V3 Attn+Btn** | **raw 67.71 / α 10.29** | **0.550** | exp011 (17주) |

→ α-MAE 기준으로는 Track B V3가 competitor들과 comparable하거나 소폭 열세.
→ DirAcc에서는 Attn+Bottleneck(0.550)이 knn_analog(0.412) 다음으로 우위.

## Step 4: 학습 데이터 규모 vs 파라미터 수

### 3-1. Ridge (mean-pooled)

| 항목 | 값 |
|------|---|
| 학습 구조 | 주별 독립 Ridge × 17 (per-week fitting) |
| 파라미터 수 | 5120 × 17 = **87,040** (bias 없음) |
| 학습 샘플 수 | 300 items |
| **샘플/파라미터 비율** | **0.003** (300 / 87,040) |

### 3-3. Attention + Linear

| 항목 | 값 |
|------|---|
| 파라미터 수 | Attention: 5,120 + Linear: 5,120×17+17 = **92,177** |
| 학습 샘플 수 | 300 items |
| **샘플/파라미터 비율** | **0.003** |

### 3-4. Attention + Bottleneck

| 항목 | 값 |
|------|---|
| 파라미터 수 (추정) | Attn: 5,120 + Bottleneck Linear: 5,120×B+B×17 (B=bottleneck dim) |
| 학습 샘플 수 | 300 items |
| **샘플/파라미터 비율** | **<<0.1** |

**샘플/파라미터 비율이 모두 10:1 미만이다 (0.003 수준).**
그러나 Ridge는 alpha 정규화로 이를 처리하므로 overfitting 자체가 문제는 아니다.
**핵심 문제는 embedding의 정보 부족 + warm/cold 분포 불일치이며, head calibration 실패가 이를 증폭시킨다.**
